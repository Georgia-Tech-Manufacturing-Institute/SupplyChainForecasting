How training works

Training runs in the background so the page stays responsive. Four stages:

  1. Load — read waterfall & consumption from the database
  2. Features — walk-forward loop over all prediction weeks (slowest step)
  3. Train — fit predictive models on the prepared data
  4. Save — write .pkl to saved_models/

Typical runtime: 2–5 min.
